# Валидация спецификаций

## Обзор

Система валидации спецификаций проверяет качество и полноту документации. Валидация учитывает статус спецификации и не блокирует коммиты для черновиков и реализованных функций.

## Использование

### Валидация одной спецификации
```bash
./bin/rake "specs:validate[054]"
```

### Валидация всех спецификаций
```bash
./bin/rake specs:validate_all
```

### Обновление статуса старых спецификаций
```bash
# Массово присвоить статус "delivered"
./bin/rake "specs:legacy_status[delivered]"

# Для индивидуальной спецификации
./bin/rake "specs:update_status[054,approved]"
```

### Комплексная проверка
```bash
./docs/.spec_status_check.sh
```

## Статусы и их влияние на валидацию

### Не блокируют коммит (ошибки показываются как предупреждения):
- **draft** - черновик, ошибки не блокируют коммит
- **implemented** - реализовано, ошибки не блокируют коммит
- **delivered** - доставлено, ошибки не блокируют коммит

### Блокируют коммит (ошибки требуют исправления):
- **business_review** - на бизнес-ревью
- **need_plan** - нужен план реализации
- **tech_review** - на техническом ревью
- **approved** - одобрено к реализации
- **in_progress** - в реализации
- **testing** - на тестировании

## Проверяемые критерии

### Обязательные секции (для активных статусов):
- ## 1. Обзор и цель
- ## 2. Требования
- ## 3. Пользовательские сценарии
- ## 4. Техническая реализация
- ## 5. Тестирование
- ## 6. Риски и зависимости
- ## 7. Внедрение

### Мета информация (предупреждения):
- **Номер:** - идентификатор спецификации
- **Название:** - наименование
- **Статус:** - текущий статус

### Качество контента:
- Минимальный размер для полноты описания

## Примеры использования

### Разработка новой спецификации:
1. Создать черновик: `./bin/rake "specs:generate[new_feature]"`
2. Валидация: `./bin/rake "specs:validate[XXX]"` (ошибки не блокируют)
3. Обновить статус: `./bin/rake "specs:update_status[XXX,business_review]"`

### Работа с существующими спецификациями:
1. Проверить все: `./bin/rake specs:validate_all`
2. Если ошибки в активных спецификациях - исправить перед коммитом
3. Черновики и реализованные функции можно коммитить с ошибками

### Коммит документации:
```bash
# Проверить статус всех спецификаций
./docs/.spec_status_check.sh

# Запустить валидацию
./bin/rake specs:validate_all

# Если блокирующих ошибок нет - коммитить
git add docs/Specs/
git commit -m "Update specification documentation"
```

## Частые проблемы

### Ошибки валидации для черновиков:
- Не блокируют коммит
- Рекомендуется исправить для качества документации

### Старые спецификации без статуса:
- Использовать `specs:legacy_status[delivered]` для массового обновления
- Или обновлять индивидуально через `specs:update_status`

### Отсутствуют обязательные секции:
- Для активных статусов - нужно добавить
- Для draft/implemented/delivered - рекомендуется добавить для полноты
