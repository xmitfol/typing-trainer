# Настройка слежения за каналами и форвардинга постов

## Обзор

Система реализована для автоматического отслеживания постов в Telegram каналах и их немедленного форвардинга пользователям.

## Компоненты

### 1. Фоновые задачи (Background Jobs)

#### MonitorJob
- **Что делает:** Периодически проверяет все активные каналы с подписками
- **Частота запуска:** Каждые 5 минут (в development - каждую минуту)
- **Очередь:** `channels`
- **Расположение:** `app/jobs/channels/monitor_job.rb`

#### FetchPostsJob
- **Что делает:** Получает новые посты из конкретного канала
- **Запускается:** MonitorJob для каждого канала
- **Очередь:** `channels`
- **Расположение:** `app/jobs/channels/fetch_posts_job.rb`

#### ProcessPostJob
- **Что делает:** Сохраняет пост в БД и запускает доставку подписчикам
- **Запускается:** FetchPostsJob для каждого нового поста
- **Очередь:** `content`
- **Расположение:** `app/jobs/content/process_post_job.rb`

#### DeliverPostsJob
- **Что делает:** Форвардит посты пользователям через Telegram API
- **Запускается:** ProcessPostJob для каждого подписчика
- **Очередь:** `content`
- **Расположение:** `app/jobs/content/deliver_posts_job.rb`

### 2. Telegram API клиент

#### ChannelFetcher
- **Что делает:** Получает посты из каналов через Telegram API
- **Особенности:** В development режиме генерирует демо-посты для тестирования
- **Расположение:** `lib/telegram_client/channel_fetcher.rb`

## Настройка

### 1. Переменные окружения

Необходимо установить в `config/application.yml` или ENV:

```yaml
telegram_bot_token: "YOUR_BOT_TOKEN"
telegram_bot_username: "YOUR_BOT_USERNAME"
```

### 2. Очереди задач

Система использует следующие очереди Solid Queue:

- **`channels`** - задачи мониторинга и получения постов из каналов
  - MonitorJob, FetchPostsJob
  - 2 потока, 1 процесс (можно изменить через CHANNELS_CONCURRENCY)
  - Интервал опроса: 1 секунда

- **`content`** - задачи обработки и доставки контента
  - ProcessPostJob, DeliverPostsJob
  - 3 потока, 2 процесса (можно изменить через CONTENT_CONCURRENCY)
  - Интервал опроса: 0.5 секунды

Конфигурация воркеров находится в `config/queue.yml`.

### 3. Запуск воркеров

```bash
# Запуск Solid Queue воркеров
./bin/rails solid_queue:start

# Или в development режиме
./bin/rails solid_queue:start -e development
```

### 4. Периодические задачи

Настроены в `config/recurring.yml`:

- **Production:** каждые 5 минут
- **Development:** каждую минуту
- **Test:** отключены

## Процесс работы

1. **MonitorJob** запускается периодически
2. Находит все активные каналы с подписками
3. Для каждого канала запускает **FetchPostsJob**
4. **FetchPostsJob** получает посты через **ChannelFetcher**
5. Для каждого нового поста запускается **ProcessPostJob**
6. **ProcessPostJob** сохраняет пост и запускает **DeliverPostsJob** для всех подписчиков
7. **DeliverPostsJob** форвардит пост пользователю

## Тестирование

В development режиме **ChannelFetcher** генерирует демо-посты с 10% вероятностью для тестирования системы.

## Ограничения

- Для реального получения постов из каналов бот должен быть добавлен в канал как администратор
- Альтернативно можно настроить webhooks для получения постов в реальном времени
- Сейчас используется симуляция для демонстрации работы

## Мониторинг

Все операции логируются в Rails logger. Уровни логирования:
- `INFO` - основные операции
- `DEBUG` - детальная информация
- `WARN` - предупреждения о недоступности каналов
- `ERROR` - ошибки при обработке
