## 3. Sequencing

**Конфликт ресурса:** и хвост P2-синка, и Ф1 тянут Алекса (фронт). Backend-ядро Ф1 с P2-фронтом НЕ пересекается.

1. **Сейчас параллельно:** Борис → Ф1-backend (F1-1…F1-8, независим, не трогает боевой data-path). Алекс → **сначала дошивает P2-синк** (useApi/saveAttempt/guest-migration с E2E).
2. **P2 вперёд админ-фронта:** P2 почти готов, даёт работающий SaaS-стейт, и `saveAttempt`-поток — естественная площадка эмиссии `events` (F3-0) для Ф3-аналитики.
3. Ф1-фронт (F1-9,10) — после P2-сшивки + готового Ф1-backend.
4. Ф2 — после Ф1-gate; stub-refund не ждёт; боевой ждёт YooKassa-shop (параллельно, не крит-путь).
5. Ф3 — эмиссию events заложить в P2-сшивку; аналитика после Ф1.
6. Ф4 — последняя.

## 4. Блокеры (владелец → митигация)
- Эмиссия events (Борис+Алекс) → заложить в P2-сшивку; скилл/деньги не ждут.
- **YooKassa-shop (PO Иван)** → только боевой refund; Ф2 целиком на stub-gate. ⚠️ верификация 1-3 дня.
- `PaymentProvider.refund` нет (Борис) → расширить Protocol, stub эмулирует.
- Бутстрап superadmin (Борис) → дешёвый CLI.
- Ресурс-конфликт фронта (Ника) → P2 сначала, админ-фронт следом.
- nginx-allowlist (инфра) → Ф4, не блок.
- **2FA до prod (Сергей/PO)** → решить на F4-SEC. ⚠️
- manual-подписка/FSM (Борис) → уточнить в Ф2.

---

## 5. Security-gate (обязательные ревью Сергея)
- **F1-SEC** — RBAC + re-auth (до денежных операций).
- **F2-SEC** — refund-путь + re-auth на деньги.
- **F4-SEC** — 🔒 вся админка (RBAC/имперсонация/re-auth/2FA/nginx). **Прод не открываем без апрува.**

---

## 6. Рекомендация — что первым
- ✅ **Борис — Ф1-backend немедленно** (независимый трек, не трогает live data-path).
- ✅ **Алекс — сначала хвост P2-синка**, затем админ-фронт Ф1 (P2 разблокирует эмиссию events для Ф3).
- ⏭️ Ф2 stub-refund — после Ф1-gate; боевой параллельно ждёт YooKassa-shop.
- ⏭️ Ф3 — эмиссию events заложить в P2; аналитику после Ф1.
- 🔒 F1-SEC → F2-SEC → F4-SEC, последний обязателен до prod.

**PO на крит-путь прямо сейчас ничего не нужно.** YooKassa-shop и решение по 2FA понадобятся к Ф2-prod / security-gate, не раньше.
