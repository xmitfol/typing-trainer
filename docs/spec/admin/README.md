# Спецификация админ-панели

Внутренний инструмент управления SaaS: поддержка юзеров, операции с подписками/платежами, продуктовая аналитика. Надстройка над готовым бэкендом (auth/billing/sync), а не новая подсистема.

- **[01_PRD.md](01_PRD.md)** — ЧТО: цели, роли (analyst/support/superadmin), матрица прав, use cases, требования по вкладкам (Обзор / Пользователи / Платежи / Статистика).
- **[02_TSD.md](02_TSD.md)** — КАК: RBAC (`users.role` + `require_admin_role`), `admin_audit_log`, refund через `PaymentProvider`, API `/admin/*`, `admin_service`, имперсонация, безопасность, тесты, rollout по фазам.

**Статус:** 🟡 Draft, требует апрува PO. Реализация — по фазам (Ф1 first), после апрува.

**Ключевые решения PO:** несколько ролей (не один флаг) · имперсонация с аудитом · возвраты через провайдера.
