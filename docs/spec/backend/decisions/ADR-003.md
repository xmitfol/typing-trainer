# ADR-003 · Family plan — read-only parental visibility

- **Status**: ✅ Accepted
- **Date**: 2026-06-06
- **Owner**: PO (Иван)
- **Related**: [PRD §8 Q2](../01_PRD.md), [TSD §4b](../02_TSD.md)

## Context

Family-подписка покрывает до 4 человек (например, родитель + 3 ребёнка) под одной оплатой 890₽/мес. Возникает вопрос: **что родитель видит про детей?**

Варианты:

1. **Полный контроль** — родитель видит всё, может писать настройки детей
2. **Read-only прогресс** — родитель видит статистику, но не лезет в настройки
3. **Только агрегаты** — родитель видит «прошёл X уроков» без детализации
4. **Ничего** — родитель только платит, дети полностью самостоятельны

С точки зрения родителя — важна mature мотивация: «я плачу за обучение детей, я хочу видеть результат». Без видимости родитель не понимает ценность подписки и churn'ит.

С точки зрения ребёнка — privacy. Особенно подросток (12-17) может негативно реагировать на «слежку».

С точки зрения compliance — COPPA-like guidelines: для kids (6-11) родитель должен иметь видимость, для подростков рекомендуется transparency «за тобой наблюдает родитель».

## Decision

**Родитель имеет read-only видимость прогресса детей. Не может менять профиль ребёнка (кроме audience при достижении возраста).** Ребёнок видит явное уведомление о том, что родитель наблюдает за прогрессом.

### Конкретно

| Что родитель ВИДИТ | Что родитель МЕНЯЕТ |
|---|---|
| Прогресс ребёнка по урокам (`GET /me/family/{id}/progress`) | ❌ |
| Историю попыток (`GET /me/family/{id}/history`) | ❌ |
| Достижения | ❌ |
| Агрегаты (дневное / недельное активное время) | ❌ |
| Профильные поля (имя, mentor, language) | Может только: `audience` (при возрасте) и `time_limit_minutes_per_day` |

### Прозрачность для ребёнка

Sub-account-юзер видит в своём Profile плашку:

> «Этот аккаунт — часть семейной подписки [Папа Иван]. Он видит твой прогресс по урокам.»

### Time limit (parental control)

Опционально parent может выставить `time_limit_minutes_per_day` для sub-account'а. Backend в `POST /me/progress` проверяет суммарное время за сегодня и возвращает `429 TIME_LIMIT_REACHED` если превышено. UI ребёнка показывает «Достиг лимита на сегодня, продолжи завтра».

## Consequences

### Что становится лучше
- Родитель видит ROI подписки → меньше churn
- Compliance с COPPA-best-practices
- Прозрачность для подростка — нет «слежки за спиной», явное уведомление
- Time-limit — реальная фича для родителей детей

### Цена
- Подросток может почувствовать дискомфорт от плашки «папа смотрит»
- Implementation cost: extra endpoints `/me/family/{id}/*` + RBAC checks
- UI complexity: parent dashboard с переключателем между sub-account'ами

### Будущие шаги / триггеры пересмотра
- После 50 family-подписок собрать feedback: реально ли родители заходят смотреть прогресс?
- Если нет — упростить (только агрегаты)
- Если да — рассмотреть push-уведомления родителю «ребёнок прошёл новый урок»
- Возможный апгрейд: «family-чат» для соревнования между членами

## Alternatives Considered

### Альтернатива А — родитель может всё писать
Отклонено: токсичный UX для подростка. Подросток не доверяет аккаунту, где «папа меняет настройки» → пишет с другого аккаунта → дополнительная churn.

### Альтернатива Б — только агрегаты
Отклонено: недостаточно для родительской мотивации. Хочется видеть «вчера прошёл лесон про знаки препинания» — это интересно и связывает.

### Альтернатива В — родитель ничего не видит
Отклонено: основная мотивация family-плана — родительский контроль. Если этого нет, нет смысла платить за family.
