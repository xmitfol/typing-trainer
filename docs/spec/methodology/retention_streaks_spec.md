# Retention / Streaks — спека v1 (MVP)

> **Статус:** ✅ к реализации · **Дата:** 2026-07-02 · **Owner:** Ника (PM) + Полина (product) · запрос PO (Иван)
> **Related:** [ROADMAP.md](../../ROADMAP.md) P4, `assets/js/dashboard.js`, `assets/js/task.js` (lesson-summary), `apiClient.getHistory`

## 1. Цель

Вернуть юзера завтра. Дать **видимую петлю привычки**: «занимаюсь каждый день». Механика v1 — **стрик** (дни подряд с занятием) + **дневная цель «10 минут»**. Без push/email (v2).

## 2. Данные (уже есть, ничего не мигрируем)

Источник — история попыток: `typing_trainer_test_history` (localStorage) ⟷ `apiClient.getHistory()` (server, `/me/history`). Каждая запись несёт `completedAt`/`created_at` + `duration`.
- **streakDays** — число последовательных календарных дней (по локальной TZ юзера) с ≥1 попыткой, оканчивающихся сегодня или вчера (сегодня-без-занятия ещё не рвёт стрик до конца дня).
- **todayMinutes** — сумма `duration` попыток за сегодня (сек→мин).
- **goalMet** — `todayMinutes >= DAILY_GOAL_MIN` (дефолт 10).
- **longestStreak** — рекорд (для мотивации), считается из той же истории.

## 3. Вычисление (клиент, fail-safe)

Новый модуль `assets/js/retention.js` (`window.retention`), чистые функции над массивом истории:
- `computeStreak(history, now)` → `{streakDays, todayMinutes, goalMet, longestStreak, activeToday}`.
- Дни группируем по `YYYY-MM-DD` локальной даты `completedAt`. Стрик: идём назад от сегодня; если сегодня нет занятия, но вчера есть — стрик ещё «жив» (показываем как есть, помечаем `activeToday=false`, подсказка «позанимайся, чтобы продлить»). Разрыв (пропущенный день) обнуляет.
- Fail-safe: пустая/битая история → `{streakDays:0, todayMinutes:0, goalMet:false, longestStreak:0}`; ничего не падает.
- Данные берём через `apiClient.getHistory()` (при useApi=true — сервер, иначе localStorage) — переиспользуем существующий адаптер.

## 4. Где показываем (v1)

- **Dashboard** (`dashboard.js`/`dashboard.html`): бейдж стрика («🔥 N дней подряд») + кольцо/бар дневной цели («X / 10 мин сегодня», зелёное при goalMet). Если стрик «жив, но не активен сегодня» — мягкая подсказка «Позанимайся сегодня, чтобы не потерять серию».
- **Экран завершения урока** (`lesson-summary`/task.js финиш): короткая строка-поощрение — «Серия: N дней 🔥» и/или «Дневная цель выполнена ✅» при переходе через порог в этом заходе (момент радости).

## 5. Out of scope v1 (→ v2)

- **Push/браузер-нотификации и email-напоминания** — отдельная инфра + privacy-ревью Сергея (PO-решение: НЕ в v1).
- Server-side стрик/агрегат (v1 считает на клиенте из истории — достаточно).
- Streak-freeze («заморозка» пропущенного дня), недельные цели, лидерборды.
- Настройка дневной цели юзером (v1 — фикс 10 мин).

## 6. Acceptance

- Стрик корректно считается на серии дат (тест-фикстура: подряд/с разрывом/сегодня-нет-вчера-есть/пусто).
- Dashboard показывает стрик + дневную цель из реальной истории; при useApi=true — с сервера, оффлайн — из localStorage (fail-safe).
- Экран финиша урока поздравляет при достижении дневной цели/продлении стрика.
- Backward-compat: гость/нет истории → нули, ничего не ломается; существующий dashboard/финиш не регрессируют.
