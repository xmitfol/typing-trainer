# Onboarding Flow - User Profile Setup

> **Назначение:** Первичная настройка профиля пользователя при первом запуске приложения
> **Phase:** Phase 1 (LocalStorage, без Backend)
> **Для кого:** Все новые пользователи
> **Версия:** 1.0
> **Дата:** 16 ноября 2025

---

## Обзор

**Onboarding Flow** — это процесс первичной настройки профиля пользователя, который запускается при первом открытии Typing Trainer. Цель — собрать базовую информацию для персонализации обучения и выбора подходящего персонажа-наставника.

### Цели онбординга:

1. **Персонализация** — обращение к пользователю по имени
2. **Подбор персонажа** — автоматический выбор наставника по полу/возрасту
3. **Адаптация советов** — рекомендации в зависимости от типа клавиатуры
4. **Вовлечение** — создание эмоциональной связи с приложением с первых секунд

### Когда запускается:

- ✅ При первом запуске приложения (LocalStorage пуст)
- ✅ После очистки данных браузера (если пользователь вернулся)
- ✅ По желанию пользователя (кнопка "Пройти онбординг заново" в настройках)

### Интерактивное поведение

**Имя (текстовое поле):**
- Плейсхолдер: "Введите имя"
- Валидация:
  - Минимум 2 символа
  - Максимум 30 символов
  - Только буквы (кириллица + латиница)
- Обязательное: Да

**Профиль (иконки с карточками):**

4 большие карточки с иконками:

| Иконка | Заголовок | Подзаголовок | Персонаж | Значение |
|--------|-----------|--------------|----------|----------|
| 👨 | Мужчина | Наставник: Анна 👩‍🏫 | anna | male |
| 👩 | Женщина | Наставник: Максим 👨‍💼 | maxim | female |
| 🤖 | Кнопыч | Робот-клавиша | knopych | child |
| 🎨 | Клавочка | Добрая помощница | klavochka | child |

**Поведение:**
- Клик на карточку → выделяется (border highlight, background change)
- Только одна карточка активна
- Обязательное: Да

**Тип клавиатуры (иконки с карточками):**

3 карточки с иконками:

| Иконка | Заголовок | Подзаголовок | Значение | Виртуальная клавиатура |
|--------|-----------|--------------|----------|------------------------|
| ⌨️ | Классическая | Обычная PC | classic | Стандартная полноразмерная |
| 💻 | Ноутбук | Компактная | laptop | Компактная (уменьшенная) |
| 🎯 | Эргономическая | Раздельная | ergonomic | Раздельная (split layout) |

**Поведение:**
- Клик на карточку → выделяется
- Только одна карточка активна
- Обязательное: Да

**Важно:** Тип клавиатуры определяет **визуальное отображение виртуальной клавиатуры** в интерфейсе:
- **Классическая** → стандартный макет (как сейчас)
- **Ноутбук** → компактная версия (меньше размер клавиш, уменьшенные отступы)
- **Эргономическая** → раздельная клавиатура (split layout с изгибом)

**Язык обучения (иконки с карточками):**

2 карточки с флагами:

| Иконка | Заголовок | Подзаголовок | Значение | Доступность |
|--------|-----------|--------------|----------|-------------|
| 🇷🇺 | Русский | QWERTY | ru | Доступно |
| 🇬🇧 | Английский | (скоро) | en | Disabled |

**Поведение:**
- Русский — выбран по умолчанию (preselected)
- Английский — disabled, opacity 0.5, cursor not-allowed

### Сохраняемые данные

После клика **"Начать обучение!"** проверяется:
- ✅ Имя заполнено (≥2 символа)
- ✅ Профиль выбран
- ✅ Тип клавиатуры выбран
- ✅ Язык выбран (по умолчанию Русский)

**Сохраняется в LocalStorage:**
- `user.name` — имя пользователя
- `user.character` — "anna" | "maxim" | "knopych" | "klavochka"
- `user.gender` — автоматически из character:
  - anna → "male" (для мужчин — женский наставник)
  - maxim → "female" (для женщин — мужской наставник)
  - knopych → "child"
  - klavochka → "child"
- `user.ageGroup` — "adult" (anna/maxim) | "child" (knopych/klavochka)
- `user.keyboardType` — "classic" | "laptop" | "ergonomic"
- `user.language` — "ru" (сейчас только русский)
- `user.onboardingCompleted` — true
- `user.createdAt` — timestamp

## Структура данных (LocalStorage)

После завершения онбординга в LocalStorage сохраняется:

```json
{
  "user": {
    "name": "Саша",
    "gender": "male",
    "ageGroup": "adult",
    "character": "anna",
    "keyboardType": "laptop",
    "language": "ru",
    "onboardingCompleted": true,
    "createdAt": "2025-11-16T23:45:00Z"
  }
}
```

**Ключи:**

| Ключ | Тип | Значения | Описание |
|------|-----|----------|----------|
| `name` | string | 2-30 символов | Имя пользователя |
| `gender` | string | "male" \| "female" \| "child" | Пол/возраст |
| `ageGroup` | string | "adult" \| "child" | Возрастная группа |
| `character` | string | "anna" \| "maxim" \| "knopych" \| "klavochka" | Выбранный персонаж-наставник |
| `keyboardType` | string | "classic" \| "laptop" \| "ergonomic" | Тип клавиатуры |
| `language` | string | "ru" \| "en" | Язык обучения |
| `onboardingCompleted` | boolean | true/false | Флаг завершения онбординга |
| `createdAt` | string | ISO 8601 | Дата создания профиля |

## UX Guidelines

### Состояния карточек выбора

**Не выбрана (default):**
- Border: 2px solid #e0e0e0
- Background: white
- Shadow: none

**Hover:**
- Border: 2px solid #2196F3
- Cursor: pointer
- Transition: 0.2s

**Выбрана (active):**
- Border: 3px solid #2196F3
- Background: #E3F2FD
- Box-shadow: 0 4px 8px rgba(33, 150, 243, 0.3)
- Checkmark icon в углу (опционально)

**Disabled (только English):**
- Opacity: 0.5
- Cursor: not-allowed
- Filter: grayscale(100%)
- Tooltip: "Скоро..."

### Кнопка "Начать обучение!"

- **Стиль:** Primary button, зеленый (#4CAF50)
- **Активна только если:**
  - ✅ Имя заполнено (≥2 символа)
  - ✅ Профиль выбран (одна из 4 карточек)
  - ✅ Тип клавиатуры выбран (одна из 3 карточек)
  - ✅ Язык выбран (по умолчанию Русский)
- **При клике:**
  1. Сохраняет профиль в LocalStorage
  2. Показывает приветствие от персонажа (3-5 сек)
  3. Переход к основному интерфейсу

**Disabled state:**
- Background: #cccccc
- Cursor: not-allowed
- Tooltip: "Заполните все поля"

### Анимации

- **Появление онбординга:** Fade-in (0.5s)
- **Клик на карточку:** Scale bounce (transform: scale(0.95) → scale(1.0), 0.2s)
- **Валидация имени:** Shake animation при ошибке (keyframes shake)
- **Финальный экран:** Confetti animation 🎉 (опционально, Phase 2)

### Доступность (a11y)

- ✅ Все поля доступны через Tab (logical tab order)
- ✅ Enter = клик на выбранную карточку / кнопку
- ✅ Space = выбор карточки
- ✅ ARIA labels для screen readers:
  - `aria-label="Выберите профиль"`
  - `aria-selected="true"` для выбранной карточки
  - `role="radio"` для карточек
- ✅ Focus-visible индикаторы (outline)
- ✅ Контрастные цвета (WCAG AA): минимум 4.5:1

### Типы виртуальной клавиатуры

**Важно:** После выбора типа клавиатуры, в основном интерфейсе отображается соответствующий макет:

**1. Классическая (classic):**
- Полноразмерная виртуальная клавиатура
- Стандартное расстояние между клавишами
- Все клавиши стандартного размера
- Макет: текущий (как в `keyboard.css`)

**2. Ноутбук (laptop):**
- Компактная виртуальная клавиатура
- Уменьшенные отступы между клавишами (~30% меньше)
- Клавиши меньшего размера (font-size: 0.85em)
- Плоский стиль (меньше shadow, меньше border-radius)
- Макет: адаптированный под ноутбук

**3. Эргономическая (ergonomic):**
- Раздельная виртуальная клавиатура (split layout)
- Два блока: левая часть + правая часть
- Небольшой изгиб/наклон для эргономики
- Расстояние между блоками (~50px)
- Макет: split keyboard (требует новой верстки)

**Реализация:**
```javascript
// Пример переключения макетов
function setKeyboardLayout(keyboardType) {
  const keyboard = document.querySelector('.virtual-keyboard');
  keyboard.classList.remove('layout-classic', 'layout-laptop', 'layout-ergonomic');
  keyboard.classList.add(`layout-${keyboardType}`);
}
```

**CSS классы:**
- `.layout-classic` — стандартный макет
- `.layout-laptop` — компактный стиль
- `.layout-ergonomic` — раздельный макет

**Приоритет реализации:**
- **P0 (Must Have):** Классическая (уже есть)
- **P1 (Should Have):** Ноутбук (CSS модификация)
- **P2 (Nice to Have):** Эргономическая (новая верстка, Phase 2)

## Edge Cases

### 1. LocalStorage недоступен

**Проблема:** Браузер блокирует LocalStorage (режим инкогнито, настройки приватности)

**Решение:**
- Показать предупреждение: "LocalStorage недоступен. Прогресс не сохранится."
- Предложить session-only режим (данные до закрытия вкладки)
- Позволить продолжить без сохранения

### 2. Пользователь закрыл вкладку во время онбординга

**Проблема:** Онбординг не завершен (частично заполнен)

**Решение:**
- При повторном открытии — восстановить частично заполненные данные
- Временно сохранять в sessionStorage по мере заполнения
- Показать онбординг заново если sessionStorage пуст

### 3. Очень длинное имя

**Проблема:** Имя не помещается в UI

**Решение:**
- Валидация: максимум 30 символов
- В UI: truncate + tooltip при наведении
  - Показывать: "Александ..."
  - Tooltip: "Александра"

### 4. Ребенок не может выбрать персонажа

**Проблема:** Застрял на выборе Кнопыч vs Клавочка

**Решение:**
- Таймаут 30 секунд → авто-выбор случайного
- Кнопка "Выбрать позже" → временно Кнопыч, можно сменить в настройках

## Implementation Notes

### Технические требования (Phase 1):

1. **HTML/CSS/JS** — без фреймворков
2. **LocalStorage API** — сохранение профиля
3. **Form Validation** — встроенная через HTML5 + JS
4. **Responsive** — работает на мобильных (хотя клавиатурный тренажер требует физическую клавиатуру)

### Приоритет реализации:

**Must Have (P0):**
- ✅ 4 шага онбординга
- ✅ Сохранение в LocalStorage
- ✅ Валидация полей
- ✅ Навигация (Назад/Далее)

**Should Have (P1):**
- ✅ Прогресс-бар
- ✅ Анимации переходов
- ✅ Приветствие от персонажа
- ✅ Настройки профиля

**Nice to Have (P2):**
- 🔜 Confetti animation
- 🔜 Аватары персонажей (иллюстрации)
- 🔜 Звуковые эффекты
- 🔜 Dark mode для онбординга
