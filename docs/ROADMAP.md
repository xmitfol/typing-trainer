## Приоритетная последовательность

### P0 — Валидация (параллельно всему)
Прогнать 3-5 реальных учеников (по одному на тир), уроки 1-10. Фиксировать где буксуют/скучают/бросают. Валидирует темп, голоса, механику. **Не блокирует разработку, но важнее новых фич.**

### P1 — Монетизация (биллинг) 🎯 *текущий фокус*
Ядро SaaS. Строим **провайдер-агностичный биллинг** (ADR-008): абстракция `PaymentProvider`, доменная модель (Plan / Subscription / статус), stub-провайдер для дев-потока, эндпоинты (checkout/webhook/status/cancel), paywall-гейт. Реального провайдера (ЮKassa/CloudPayments/Stripe) подставляем позже без переписывания. Фронт: страница тарифов → checkout, показ статуса подписки.
- **Сейчас:** дизайн + scaffold (ADR-008 + модели + интерфейс + stub + эндпоинты).

### P2 — OAuth + синк прогресса (Sprint 2)
- OAuth Yandex/VK (ADR-007 готов): `core/oauth.py` → `/auth/oauth/*/start`+`/callback`; guest-migration.
- Синк прогресса localStorage ↔ сервер (аккаунт держит прогресс/подписку). apiClient-адаптер уже есть.

### P3 — Онбординг-редизайн
По фидбэку PO (форма конфлейтит юзера/наставника/возраст). Ждёт макета от PO.

### P4 — После первых данных
- Адаптивный движок «слабые клавиши» (спек `docs/spec/methodology/adaptive_repetition_spec.md`).
- Петля удержания: стрики, «10 минут в день», напоминания.
- Контент-QA: вычитка читабельности 448 уроков.

### Инфра (сквозное)
- **CI с реальным Docker-прогоном стека** (урок Sprint 1: 2/3 багов прошли бы зелёный unit-CI).
