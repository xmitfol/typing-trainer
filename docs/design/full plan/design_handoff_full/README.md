# typing-trainer · Full Design Handoff

Полный дизайн-пакет нового интерфейса тренажёра слепой печати. Содержит **9 экранов / флоу** в виде интерактивных HTML+React-прототипов, плюс компонент эргономической клавиатуры с 3 типами раскладки.

Этот документ — план поэтапного внедрения дизайна в production-стек проекта.

## Phase 0 · Foundation (1–2 дня)

**Цель:** заложить токены и примитивы, которые используют все остальные фазы.

### Что делать

1. **Установить шрифты** через Google Fonts (см. выше) или self-host.
2. **CSS токены** — добавить переменные в глобальный стиль (`globals.css` / `tokens.css`).
3. **Базовые компоненты:**
   - `<Button variant="primary|secondary|ghost" />` — три варианта, см. примеры в `auth.jsx` (PrimaryButton), `landing.jsx`, `pricing.jsx`
   - `<Card />` — карточка с border + radius 14-18px
   - `<Input label hint error suffix />` — стандартное поле, см. `auth.jsx` (Input)
   - `<Tabs items active onChange />` — табы с подчёркиванием активного, см. `auth.jsx`, `lesson.jsx` (ProfileTabs)
   - `<Modal />` — затемнённый overlay с центрированной карточкой, см. `task.jsx`, `pricing.jsx`
4. **Перенести `portraits.jsx`** как компоненты `<UserPortrait />` и `<MentorPortrait />` + хелпер `detectGender(name)`.

### Acceptance

- Тёмная тема работает через `data-theme="dark"` на `<html>` или `<body>`
- Кнопки/инпуты соответствуют визуалу референсов

## Phase 1 · Keyboard Component (3–4 дня)

**Цель:** core-компонент `<Keyboard />` с тремя типами раскладки и палитрой пальцев.

### Файлы для изучения

- `reference/keyboard/keyboard.jsx` — ВЕСЬ исходник; самый важный файл пакета
- `reference/keyboard/app.jsx` — пример встраивания с практикой

### Что делать

1. **Структура данных** (ROWS / NUMPAD / NAV / CLASSIC_BOTTOM / THUMB_ROWS) — перенести как есть. Это **единый источник правды**: один и тот же массив рендерится для Classic / Laptop / Ergonomic.

2. **Компонент `<Key />`** с props:
   - `data: { l, u?, f, w?, home?, mod?, code?, alt? }`
   - `unit` — базовый размер в px (60 desktop, 48 laptop, 38 tablet, 22 mobile)
   - `state: 'default' | 'hover' | 'active' | 'highlight' | 'error'`
   - `intensity: 'full' | 'strip' | 'highlight'`
   - `dark`, `modifierStyle: 'solid' | 'hatched'`

3. **`<ClassicKeyboard />`** — alpha-блок + nav-кластер + numpad. **Критично:** используется алгоритм `padRow` для выравнивания правого края всех 5 рядов (см. функцию в keyboard.jsx). Без этого Backspace «торчит» за пределы.

4. **`<ErgoKeyboard />`** — две `<KeyboardHalf />`, каждая повёрнута `transform: rotate(±angle)` относительно нижнего внутреннего угла. Split-Space — две независимые половины purple.

5. **`<MobileErgoKeyboard />`** — упрощённая для viewport < 480px, без угла, половины почти стыкуются.

### API

```jsx
<Keyboard
  type="classic" | "laptop" | "ergonomic"
  layout="standard" | "phonetic" | "typewriter" | "mac"  // RU input variants
  intensity="full"                                        // 'full' | 'strip' | 'highlight'
  activeKey="KeyF"                                        // подсветка нажатой
  highlightChar="а"                                       // подсветка следующей
  errorKey={null}                                         // последняя ошибка
  showFingerHint                                          // включать ли подсказку пальца
  dark={false}
/>
```

### Acceptance

- 3 типа раскладки рендерятся корректно (см. `keyboard/index.html`)
- Правый край всех рядов выровнен (Backspace, \\, Enter, Shift, Ctrl — на одной линии)
- Numpad: + и Enter высотой 2 ряда, 0 шириной 2 клавиши
- Стрелки центрируются в клавише, не прижаты в угол
- Home-row bumps на А и О
- Цвета пальцев точно из палитры

### Зависимости: Phase 0

## Phase 2 · Auth + Onboarding (2–3 дня)

**Цель:** дать пользователю войти и быстро настроить курс.

### Auth (auth.jsx)

Один экран с тремя табами:
- **Войти** — email + пароль + чекбокс «Запомнить» + Войти + соц.
- **Регистрация** — соц. ПЕРВЫЕ + email + пароль с password-strength bar + чекбокс согласия + Создать аккаунт
- **Забыли пароль?** — magic link (без формы сброса с кодом — отправляем ссылку, по клику логиним сразу)
- **Email verification** — НЕ блокирует, есть кнопка «Продолжить без подтверждения»

#### Что важно

- **Минимум полей при регистрации** (email + пароль). Имя, возраст, наставник — собирается в онбординге, НЕ здесь.
- Соц. провайдеры показываем ПЕРЕД email-формой на регистрации (~30% пользователей выберут).
- reCAPTCHA — invisible v3, никаких чекбоксов «я не робот».
- Возможные провайдеры: ВКонтакте, Яндекс ID, Google, Telegram. Одноклассники — опционально.

### Onboarding (onboarding.jsx · Variant 3 «Минимальный»)

После регистрации:

1. Имя + автодетект пола по окончанию (`detectGender(name)`) с возможностью переключить М/Ж.
2. **«Кто будет учиться?»** — 3 карточки: Взрослый (18+) / Подросток (12-17) / Ребёнок (6-11). Аватары в карточках меняются по полу.
3. **Наставник:**
   - Если **Взрослый** — выбор Anna или Maxim (по стилю обучения, не по полу пользователя)
   - Если **Подросток/Ребёнок** — фиксирован (Кнопыч / Клавочка), показывается как пресет
4. **Параметры (язык, клавиатура, тема)** — детектим **тихо в фоне** по системе/браузеру и UA. Без отдельной плашки на экране онбординга. Если пользователь захочет поменять — есть Настройки.

### Acceptance

- Гендер-детект работает на распространённых русских именах (Анна → ж, Алексей → м, Илья → м-исключение)
- Аватары переключаются live при изменении имени
- Кнопка финиша знает имя: «Начать обучение, Анна →»
- Регистрация занимает **<30 секунд**

### Зависимости: Phase 0

## Кросс-фазные паттерны

### Mentor system

Каждый пользователь имеет одного из 4 наставников (выбор в онбординге для adult, фикс для teen/kid):

| ID | Имя | Тон |
|---|---|---|
| `anna` | Анна | Мягкий темп, подробные объяснения, поощряющий |
| `maxim` | Максим | Энергичный, по делу, упор на скорость |
| `knopych` | Кнопыч | Гейминг, мемы, теги «крут!», «давай ещё!» |
| `klavochka` | Клавочка | Сказочный, ласковый, «молодец, малыш» |

**Контент-стратегия:** все фразы (приветствия в уроке, советы в task screen, поздравления при выполнении) должны быть заготовлены в **четырёх тональностях**. Это работа контент-команды, не разработчиков. Backend выдаёт `mentor.id` и template ID, фронт подставляет.

### Auto-detection (тихий фон)

Не отвлекаем пользователя на онбординге, детектим:
- **Язык:** из `navigator.language`, `Accept-Language` header
- **Клавиатура:** UA-sniffing (Mac → ergo более вероятна, Windows desktop → classic, mobile → on-screen). Точное определение numpad невозможно — доверяем UA.
- **Тема:** `prefers-color-scheme`

Если нужно изменить — пользователь идёт в Настройки. На онбординге это **не должно занимать экран**.

## Тесты, которые надо запустить

1. **Раскладка ЙЦУКЕН → пальцы:** для каждой клавиши проверить, что её цвет совпадает с пальцем по методике слепой печати. Готовый матчинг — в `keyboard.jsx` (массив ROWS).
2. **Right-edge alignment:** в Classic клавиатуре все 5 рядов должны заканчиваться на одной вертикальной линии. Проверять при разных unit (60, 48, 38).
3. **Gender detection accuracy:** на тестовом списке из 100 имён должно быть >95% корректных определений.
4. **Mentor portraits на dark theme:** портреты на тёмном фоне читаемы? (фон портретов pastel — может конфликтовать)
5. **Mobile viewport (420px):** все экраны масштабируются?

## Открытые вопросы для продукта

1. **Финальные иллюстрации персонажей** (Кнопыч, Клавочка) — нужны от иллюстратора как маскоты бренда. Текущие SVG — temporary.
2. **Реальные фото или иллюстрации** для аватаров взрослого/подростка/ребёнка? Сейчас SVG-портреты.
3. **Контент-фразы наставников** (приветствия, советы, поздравления) в 4 тональностях — заказ контент-команде.
4. **Цены подписки** — текущие 490/890 ₽/мес и градации скидок (0/0/15/25/35%) согласовать с маркетингом.
5. **«Trial 1 неделя»** платный (150 ₽) или бесплатный? Платный безопаснее для unit economics, бесплатный лучше для конверсии.

## Технические рекомендации

- **Стек React + CSS Modules** или **Tailwind** — оба подходят. Не рекомендую CSS-in-JS из перфоманс-соображений (клавиатура содержит >100 клавиш).
- **Состояние:** Zustand или Redux для глобального (профиль, текущий курс, наставник). Локальное состояние компонента — useState.
- **Маршрутизация:** Next.js (если SSR нужен для лендинга) или React Router.
- **API контракты:** опишу отдельно при необходимости. Основные сущности: User, Profile, Mentor, Course, Module, Lesson, Exercise, Attempt, Subscription.
