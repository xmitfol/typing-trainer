# typing-trainer · Keyboard + Task Screen · Handoff для верстальщика

Готовый vanilla-стек экрана упражнения (`task.html`) с Web Component клавиатуры. **Никакого build-step, никакого React** — открывается через `file://` или любой статик-сервер.

---

## Структура

```
keyboard_handoff/
├── task.html                       ← ЭКРАН УПРАЖНЕНИЯ (главный, готов к интеграции)
├── poc.html                        ← демо-песочница клавиатуры (для проверки всех режимов)
└── assets/
    ├── css/
    │   ├── tokens.css               — дизайн-переменные (палитра, типографика, dark theme)
    │   └── base.css                 — примитивы (.btn, .card, .field, .tabs, .modal)
    └── js/
        ├── typing-keyboard.js       — <typing-keyboard> Web Component (ЯДРО)
        ├── task.js                  — логика экрана: ввод, метрики, состояния, success
        ├── lesson-data.js           — данные уроков (target-тексты, советы наставника)
        ├── portraits.js             — SVG-портреты наставников + detectGender()
        └── ui-primitives.js         — template-функции (для остальных экранов)
```

---

## Как запустить

```bash
cd keyboard_handoff
python3 -m http.server 8000
# открыть http://localhost:8000/task.html?tier=tier1&lesson=1
```

⚠️ **Именно через сервер**, не через `file://` напрямую — иначе `task.js` не сможет прочитать query-параметры на некоторых браузерах и модульная загрузка скриптов будет капризничать. Любой статик-сервер подойдёт.

## 1. Web Component `<typing-keyboard>`

Ядро. Регистрирует custom element. Один тег рисует полную клавиатуру.

### Подключение

```html
<script src="assets/js/typing-keyboard.js"></script>
<typing-keyboard id="kb" type="classic" unit="40"></typing-keyboard>
```

### Атрибуты (реактивные — меняешь атрибут, перерисовывается сам)

| Атрибут | Значения | Default | Назначение |
|---|---|---|---|
| `type` | `classic` / `laptop` / `ergonomic` | `classic` | Тип клавиатуры |
| `unit` | px | `56` | Базовый размер клавиши. Вся геометрия от него |
| `intensity` | `full` / `strip` / `highlight` | `full` | Насыщенность подсветки пальцев |
| `modifier-style` | `solid` / `hatched` | `solid` | Заливка модификаторов |
| `theme` | `light` / `dark` | inherited | Тема |
| `angle` | degrees | `14` | Угол разворота половин (только ergonomic) |
| `gap` | px | `96` | Разрыв между половинами (только ergonomic) |
| `active-key` | KeyCode (`KeyF`) | — | Клавиша в состоянии «нажата» |
| `highlight-char` | один символ (`а`) | — | Подсветка «следующей нужной» клавиши |
| `error-key` | KeyCode | — | Клавиша мигает красным |

### JS API

```js
const kb = document.querySelector('#kb');

kb.flashActive('KeyF');           // зелёная вспышка ~160мс (правильное нажатие)
kb.flashActive('KeyF', 200);      // с кастомной длительностью
kb.flashError('KeyG');            // красная вспышка ~280мс (ошибка)
kb.setAttribute('highlight-char', 'а');   // подсветить след. клавишу
kb.removeAttribute('highlight-char');     // убрать подсветку
kb.setAttribute('type', 'ergonomic');     // сменить тип
```

### Как выбрать `unit` под разные типы и экраны

Клавиатура рисуется в натуральную ширину от `unit`. Подбирай так, чтобы влезала в контейнер:

| Тип | Ширина | Рекоменд. unit (desktop) |
|---|---|---|
| Classic (alpha+nav+numpad) | ~24×unit | **40** |
| Laptop (alpha+стрелки) | ~16×unit | **46** |
| Ergonomic (половины+nav+numpad) | ~27×unit | **38** (+ `gap=52`, `angle=10`) |

В `task.js` это уже зашито в обработчике смены типа — смотри функцию переключения `type-select`. Под планшет/мобайл уменьшай unit пропорционально (или оберни стейдж в `transform: scale()` по `clientWidth / kbWidth`).

### Изоляция

Компонент использует Shadow DOM — внутренние стили не текут наружу и наоборот. **Важно:** скриншот-тулзы на основе html-to-image НЕ видят содержимое Shadow DOM (клавиатура выйдет пустой), но в реальном браузере всё рисуется. Проверять — живым браузером.

### Что внутри (если надо менять раскладку)

- `ROWS` — 4 ряда буквенного блока, разбиты на `{ left, right }`. Каждая клавиша: `{ l: 'А', u: '1'?, f: 'blue', w: 1.25?, home: true?, mod: true?, code: 'KeyF' }`
- `CLASSIC_BOTTOM` — нижний ряд (Ctrl/Win/Alt/Space/...)
- `THUMB_MIN` — нижний ряд эргономики (split-Space)
- `NUMPAD`, `NAV_TOP` — цифровой блок и навигация
- `padRows()` — **критичная функция**: выравнивает правый край всех 5 рядов на одну линию (компенсирует разное число клавиш в ряду). Не удалять.

## 2. Экран упражнения `task.html`

### URL

```
task.html?tier=tier1&lesson=1
```

`task.js` читает `tier` и `lesson` из query, берёт данные урока из `lesson-data.js`.

### Структура экрана

```
┌─────────────────────────────────────────── task-card ──┐
│                                  [наставник + bubble] ↗ │  ← mentor, overflows top
│  ┌── task-body (grid 180 / 1fr / 220) ───────────────┐ │
│  │ 1.1        │ progress + target text │  ВРЕМЯ      │ │
│  │ (попытка)  │ [печатаешь здесь]      │  СКОРОСТЬ   │ │
│  │ легенда    │                        │  ТОЧНОСТЬ   │ │
│  │ пальцев    │                        │            │ │
│  └────────────────────────────────────────────────────┘ │
│  [toolbar: палитра / скрыть / звук / метроном / layout]  │
│  ┌── kb-stage ──────────────────────────────────────┐   │
│  │           <typing-keyboard>                       │   │
│  └──────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

### Два состояния (переключаются в `task.js`)

1. **Active** (`#task-body` видим) — пользователь печатает. Live: прогресс-бар, время, скорость (зн/мин), точность. Текущий символ в `#target` подсвечен, соответствующая клавиша — через `highlight-char`.
2. **Success** (`#success.show`) — после завершения. Заголовок «Всё верно.» (Serif), оценка ★, финальные метрики, портрет наставника + конфетти, кнопки «Повторить» / «Продолжить».

Переключение: `task.js` добавляет/убирает класс `.show` на `#success` и `.hide` на `#task-body`.

### Ввод

Скрытый `<input class="capture">` держит фокус. На `keydown`:
- символ совпал с ожидаемым → `kb.flashActive(e.code)`, продвигаем позицию
- не совпал → `kb.flashError(e.code)`, считаем ошибку
- `highlight-char` обновляется на след. символ после каждого нажатия

Вся логика — в `task.js`, функция обработки `keydown`. Метрики (WPM, точность, время) считаются там же — чистый JS, без зависимостей.

### Toolbar

- **Палитра** — индикатор режима цвета (по умолч. активен)
- **Скрыть индикатор набора** (`#hide-hint-btn`) — убирает подсветку текущей клавиши для тренировки вслепую. Тоггл снимает `highlight-char`.
- **Звук / Метроном** — заглушки, подключи свои SFX к тому же `keydown`-листенеру
- **Layout select** — Стандартная / Фонетическая / Машинопись / МАС (раскладка ВВОДА)
- **Type select** — Полноразмерная / Ноутбук / Эргономическая (меняет `type` + подбирает `unit`)

## 3. Маршрутизация уроков (важно!)

```
course.html / dashboard.html
   │ клик по уроку
   ▼
lesson.html?tier=tier1&lesson=N      ← ТЕОРИЯ (обучающий текст), ВСЕГДА первая
   │ кнопка «Начать упражнение»
   ▼
task.html?tier=tier1&lesson=N        ← ПРАКТИКА (этот экран)
   │ success → «Продолжить»
   ▼
lesson.html?tier=tier1&lesson=N+1    ← следующий урок, снова с теории
```

⚠️ **`task.html` — никогда не точка входа в урок напрямую из списка.** Сначала всегда `lesson.html` (теория), оттуда кнопка ведёт в `task.html`. И dashboard, и course ведут на `lesson.html`, не на `task.html`.

## 4. Данные урока (`lesson-data.js`)

Структура — массив/объект уроков по tier. Каждый урок:
```js
{
  tier: 'tier1', lesson: 1,
  num: '1.1',
  target: 'фыва олдж ...',     // текст для набора
  mentorTip: '...',             // совет в bubble
  mentorHint: '...',            // вторая строка
  // ... оценки, пороги
}
```
Расширяй массив новыми уроками — task.js подхватит по `?lesson=N`.

## 5. Критические правила (НЕ менять)

1. **Палитра пальцев** — 6 цветов в `tokens.css` (`--f-pink`…`--f-purple`). Фиксированы (пользовательская инструкция). В Shadow DOM компонента продублированы как `--kb-*`.
2. **Раскладка ЙЦУКЕН — деление рук по палитре**: левая Й Ц У К Е / Ф Ы В А П / Я Ч С М И; правая Н Г Ш Щ З Х Ъ / Р О Л Д Ж Э / Т Ь Б Ю. Home keys: **А** (`KeyF`) и **О** (`KeyJ`) — с тактильными bump-индикаторами.
3. **Split-Space на эргономике** — две независимые половины purple (зависимость от спеки 008).
4. **Эргономика**: половины развёрнуты на угол + numpad/nav стоят **вертикальными блоками справа** (как classic), основной блок приподнят на 30px относительно блоков.
5. **Стек**: только HTML+CSS+JS. ❌ React/Vue, ❌ npm/build, ❌ TypeScript, ❌ препроцессоры.
