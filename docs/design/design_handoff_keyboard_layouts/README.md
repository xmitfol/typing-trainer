# Handoff: Клавиатурные раскладки для тренажёра слепой печати (spec 009)

## Обзор

Дизайн визуальных клавиатур для проекта **typing-trainer** — учебный тренажёр слепой печати для русской раскладки ЙЦУКЕН. Дизайн покрывает **три типа клавиатур**, между которыми пользователь сможет переключаться в зависимости от своего железа:

1. **Classic** — полноразмерная клавиатура (alpha-блок + nav-кластер + numpad)
2. **Laptop** — компактная без numpad, со стрелками в нижне-правом углу
3. **Ergonomic** — раздельная (split) клавиатура с разворотом половин и split-Space

Каждая клавиша окрашена в цвет пальца, который её обслуживает по методике слепой печати. Палитра пальцев — **6 фиксированных цветов из `APP_CONFIG.keyboard.fingerColors`** — это пользовательская инструкция, цвета менять нельзя.

## Архитектура

### Источник правды для клавиш

В `reference/keyboard.jsx` все клавиши описаны как декларативные структуры данных:

- **`ROWS`** — 4 строки буквенного блока (числа, ЙЦУКЕН, ФЫВА, ЯЧСМИ), разделённые на `{ left, right }`
- **`THUMB_ROWS`** — нижние ряды для эргономической клавиатуры (`minimal` и `full` варианты thumb-кластера)
- **`CLASSIC_BOTTOM`** — нижний ряд для classic/laptop (Ctrl/Win/Alt/Space/...)
- **`NUMPAD`** и **`NAV`** — цифровой блок и навигация для classic

**ВАЖНО**: одна и та же структура данных рендерится для всех трёх раскладок. При переносе обязательно сохрани этот принцип — единый источник правды → 3 layout-рендера.

### Формат описания клавиши

```js
{
  l: 'А',        // основной символ (отображаемая метка)
  u: '1',        // вторая метка (Shift-символ) — необязательно
  f: 'blue',     // палец/цвет: pink|orange|green|blue|indigo|purple
  w: 1.25,      // ширина в условных единицах "u" (по умолчанию 1)
  home: true,    // home-row bump indicator (точка под буквой)
  mod: true,     // модификатор (Tab, Shift, Caps, Enter, ⌫, Ctrl, ...)
  code: 'KeyF',  // physical key code (для матчинга нажатий)
  alt: '1',      // альтернативный символ для матчинга
}
```

## Состояния клавиш

| State | Визуал | Когда |
|---|---|---|
| `default` | Базовая заливка цвета пальца (или нейтральный для модификатора) | Покой |
| `hover` | Outline 1.5px solid цветом пальца | Курсор над клавишей |
| `active` | Полная заливка + glow shadow `0 0 0 6px <color>33` | Клавиша нажата правильно |
| `highlight` | Outline 2px + диффузный glow `0 0 18px <color>66` | Подсказка: эту клавишу нужно нажать следующей |
| `error` | Полная заливка `#ff4757` + красный glow | Нажата не та клавиша |

Реализация состояний — в `keyboard.jsx`, функция `Key`, переменные `shadow`/`bg`/`labelColor`.

## Раскладка / Геометрия

### Алгоритм выравнивания краёв (Classic / Laptop)

Каждый ряд клавиш имеет разное число клавиш и разную сумму ширин. Чтобы **правый край всех 5 рядов лёг на одну вертикальную линию**, считаем максимальную ширину среди всех рядов (key widths + gap widths) и докидываем "дефицит" каждому короткому ряду на специально назначенные клавиши:

- **Ряд 0** — самый широкий, без подгонки
- **Ряд 1** — дефицит на `Tab` (так `\` остаётся 1u)
- **Ряд 2** — дефицит поровну на `Caps` и `Enter` (остаются равными)
- **Ряд 3** — дефицит поровну на оба `Shift` (остаются равными)
- **Ряд 4** (Ctrl/Win/Alt/Space/...) — весь дефицит уходит на `Space` (он и должен быть variable)

Точная формула — в функции `padRow` в `keyboard.jsx`. **Это критично для визуального качества — без этого правый край "плавает"**.

### Numpad — CSS Grid

Numpad использует CSS Grid 4×5 с `gridRow: 'span 2'` для tall-клавиш `+` и `Enter`. Клавиша `0` имеет `gridColumn: '1 / span 2'` (ширина 2u). Цвета:

- Col 1 (`7`/`4`/`1`): **blue** (указат.)
- Col 2 (`/`, `8`, `5`, `2`): **green** (средний). `5` — home key (bump)
- Col 3 (`*`, `9`, `6`, `3`, `,`): **orange** (безымянный)
- Col 4 (`–`, `+`): **orange** (безымянный тянется к краю)
- `NumLock`, `Enter` — нейтральные модификаторы (без цвета пальца)
- `0` — **purple** (большой)

### Ergonomic — разворот половин

Левая и правая половины отрисовываются отдельно (компонент `KeyboardHalf` для каждой стороны), и каждая поворачивается через CSS `transform: rotate(±angle deg)` относительно нижнего внутреннего угла:

```css
.half-left  { transform: rotate(-14deg); transform-origin: bottom right; }
.half-right { transform: rotate(+14deg); transform-origin: bottom left;  }
```

Между половинами — `gap: 96px` (для подставки под запястья). Угол и gap должны быть **настраиваемыми параметрами** (см. Tweaks ниже).

**Split-Space**: пробел разделён на две независимые половины (по одной на каждой стороне). Цвет — `purple` (большой палец). Это обязательное требование из спеки 008.

### Размеры (unit) под разные viewport'ы

| Viewport | Width | Unit (px) | Где применять |
|---|---|---|---|
| Desktop | 1400 | 60 | Веб, основной |
| Laptop  | 1180 | 48 | Узкие ноутбуки |
| Tablet  | 820  | 38 | iPad-планшеты |
| Mobile  | 420  | 22 | Мобильный, без угла (половины butted) |

В мобильной версии эргономика без разворота: половины стыкуются почти вплотную, чтобы поместиться, но цвета и логика рук сохраняются. Используется отдельный компонент `MobileErgoKeyboard`.

**Scale-to-fit**: при ширине окна < native viewport — клавиатура целиком масштабируется через `transform: scale()`, сохраняя пропорции и не теряя качества (потому что это математическое масштабирование, а не пересчёт unit).

## Раскладка ЙЦУКЕН — деление на руки

**ВАЖНО**: бриф 009 содержал текст с ошибочным разделением рук (`Н/Р/Т` на левой стороне). Я следовал **палитре пальцев** (это пользовательская инструкция, она первична):

**Левая половина**: `Й Ц У К Е` / `Ф Ы В А П` / `Я Ч С М И` + цифры `Ё 1 2 3 4 5` + `Tab Caps Shift Ctrl`

**Правая половина**: `Н Г Ш Щ З Х Ъ` / `Р О Л Д Ж Э` / `Т Ь Б Ю .` + цифры `6 7 8 9 0 - =` + `Backspace \ Enter Shift Ctrl`

Home keys: **А** (левый указат., код `KeyF`) и **О** (правый указат., код `KeyJ`). На них рисуется тактильный bump-indicator (точка снизу клавиши).

## Tweaks (для дизайнерского изучения; в production оставить только нужное)

В референсном прототипе есть панель настроек:

- **layout**: `classic` | `laptop` | `ergonomic` — переключатель раскладки **(оставить — это пользовательский setting)**
- **dark**: `true` | `false` — тёмная тема **(оставить — настройка пользователя)**
- **intensity**: `full` | `strip` | `highlight` — насыщенность подсветки пальцев **(опционально — можно вынести в setting "intensity of color hints")**
- **modifierStyle**: `solid` | `hatched` — заливка модификаторов **(дизайнерский референс, не обязательно в проде)**
- **angle**, **gap**, **geometry** (`staggered` / `ortho`), **thumbCluster** (`minimal` / `full`) — параметры эргономической **(можно зафиксировать defaults: angle=14°, gap=96px, geometry=staggered, thumbCluster=minimal)**

- Upper character (вторая метка над основной): `unit × 0.2` px, opacity 0.55

## Радиусы, тени, переходы

```css
/* Key */
border-radius: calc(var(--unit) * 0.14);     /* 8.4px при unit=60 */
border: 1px solid var(--border-color);

/* Light shadow */
box-shadow:
  0 1px 0 rgba(255,255,255,0.9) inset,
  0 1px 2px rgba(0,0,0,0.06),
  0 2px 6px rgba(0,0,0,0.04);

/* Dark shadow */
box-shadow:
  0 1px 0 rgba(255,255,255,0.04) inset,
  0 2px 4px rgba(0,0,0,0.5);

/* Active state shadow */
box-shadow:
  0 0 0 2px var(--finger-color),
  0 0 0 6px rgba(<finger>, 0.2),
  0 4px 12px rgba(<finger>, 0.4);

/* Transitions */
transition: all 0.15s ease;

/* Key hover lift */
.key:hover { transform: translateY(-1px); }
```

Гэп между клавишами: `unit × 0.08` px

## Mobile mockup особенности

- Без угла разворота (`angle: 0`)
- Половины стыкуются с минимальным зазором (`unit × 0.25`)
- `MobileErgoKeyboard` принимает `activeHand` (`'left' | 'right' | null`) и **затемняет неактивную руку** при печати — помогает учить какую руку использовать
- Practice text использует `word-break: normal` и `overflow-wrap: break-word` (важно для длинных русских слов)
