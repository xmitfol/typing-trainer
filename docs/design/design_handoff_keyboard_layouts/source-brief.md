## 📋 Краткое описание

В онбординге у нас есть три варианта клавиатуры: **Classic** (десктоп, full-size), **Laptop** (без numpad/nav) и **Ergonomic** (раздельная). Первые две реализованы — для эргономики нужен полноценный визуальный дизайн раздельной клавиатуры, поскольку существующая стандартная QWERTY-разметка для неё не подходит.

В текущем коде вариант "Эргономическая" в онбординге **отключён** как «скоро» (`<div class="selection-card disabled">`). Этот ТЗ нужен чтобы разблокировать опцию, как только дизайн будет готов.

---

## 🎯 Контекст и почему это нужно

**Проект:** SaaS тренажёр слепой печати на русской раскладке ЙЦУКЕН. Виртуальная клавиатура — центральный UI-элемент: показывает пользователю **какой палец какой какой клавиши касается** через цветовое кодирование. См. макеты текущей классической клавиатуры в [docs/assets/виртуальная клавиатура.jpg](../assets/виртуальная%20клавиатура.jpg) и финальный кадр приложения в [docs/assets/интерфейс.jpg](../assets/интерфейс.jpg).

**Зачем эргономика:**
- Часть пользователей физически использует Microsoft/Logitech Ergo / Kinesis / ZSA Moonlander / split-кастомы
- Раздельная клавиатура физически **разделяет руки**, делает невозможным «дотянуться правой рукой до Z» — это часть правильной техники
- Visual mapping должен соответствовать **физической** клавиатуре пользователя, иначе подсветка пальцев становится лжива

---

## 👥 Целевая аудитория для эргономической раскладки

**Primary:** Программисты и тех-специалисты, инвестировавшие в эргономическую периферию (предположительно <5% базы, но самые ценные — высокая платёжеспособность, daily-use).

**Secondary:** Пользователи с RSI / туннельным синдромом, переходящие на эргономику по медицинским показаниям.

## 📐 Что нужно нарисовать

### Обязательно

#### D-1: Базовый split-layout раздельной клавиатуры

**Описание:** Раздельная (split) раскладка с двумя «половинами» — левой и правой — расположенными с **угловым разворотом ~15°** друг от друга (как у MS Sculpt Ergo / Kinesis Freestyle). Между половинами явный визуальный gap (~80-120px по дизайну).

**Включает:**
- Левая половина: цифры 1-6, ряды ЙЦУКЕН → ФЫВАПР → ЯЧСМИТ, левый Shift, Tab, CapsLock, левый Ctrl/Alt, **левая половина split-Space**
- Правая половина: цифры 7-0 и -=, клавиши ГНШЩЗХЪ\ → ОЛДЖЭ → ЬБЮ. → Enter и правый Shift, правый Ctrl/Alt, **правая половина split-Space**
- Между руками: пустая зона или иллюстрация подставки для запястий

**Acceptance Criteria для дизайнера:**
- [ ] Каждая буква на правильной половине (не «дотянуться левой до правой стороны»)
- [ ] Цветовая раскладка пальцев СОХРАНЕНА (см. ниже палитру)
- [ ] Split-Space из специ-006 [008_Split_Space_Bar_Specification.md](008_Split_Space_Bar_Specification.md) интегрирован — две независимые половины Space, по одной под каждым большим пальцем
- [ ] Общая ширина двух половин + gap ≈ ширине существующей классической клавиатуры (для встраивания в тот же layout slot)
- [ ] Высота не больше существующей classic + 20% (чтобы не ломать вёрстку нижней панели)

---

#### D-2: Mobile/laptop-вариант (упрощённый)

**Описание:** На разрешениях <768px показать **только две половины**, без gap между ними (или совсем без раздела), но с сохранённой цветовой схемой. Это compromise для мобильных — реальную эргономику на мобильном не симулируешь, но обозначение «вы выбрали ergo» должно остаться визуально.

**Acceptance Criteria:**
- [ ] Помещается в viewport 360px-768px
- [ ] Сохранено цветовое кодирование пальцев
- [ ] (Опц.) Показать **только текущую активную руку** при наборе

---

### Желательно (если есть бюджет)

#### D-3: Hover-состояние «подставка для большого пальца»

Между половинами — маленькая зона thumb cluster для дополнительных клавиш (Backspace, Enter, modifiers). На многих современных ergo-клавиатурах есть отдельный кластер под большим пальцем.

#### D-4: Анимация «привыкаешь к раздельной»

При первом запуске эргономического режима — короткая анимация развоздящихся половин (классическая → ergo). Чисто для wow-эффекта, не критично.

## 🎨 Визуальная палитра — палитра пальцев (СТРОГО соблюдать!)

Цветовая система кодирует **какой палец нажимает клавишу**. Эти цвета определены в [config/settings.js](../../config/settings.js) → `APP_CONFIG.keyboard.fingerColors` и используются по всему приложению:

| Цвет | Hex | Палец | Где применяется |
|------|-----|-------|-----------------|
| 🌸 Розовый | `#ff7675` (gradient до `#e84393`) | Мизинец (обе руки) | Ё, Й, Ф, Я, ё/1, ъ/=, э, ., -, |
| 🍑 Оранжевый | `#fdcb6e` (до `#f39c12`) | Безымянный | Ц, Ы, Ч, 2, 9 |
| 💚 Зелёный | `#00b894` (до `#00a085`) | Средний | У, В, С, 3, 8 |
| 🌊 Голубой | `#74b9ff` (до `#0984e3`) | Указательный левый | К/Е, А/П, М/И, 4/5 |
| 🌀 Синий | `#0984e3` (до `#2d3436`) | Указательный правый | Н/Г, Р/О, Т/Ь, 6/7 |
| 💜 Фиолетовый | `#a29bfe` (до `#6c5ce7`) | Большой (Space) | Space (обе половины) |

**Эти цвета — не визуальный декор, а пользовательская инструкция.** Изменение палитры ломает обучающую методику.

## 🔧 Технические ограничения

Дизайнер должен учитывать (но НЕ реализовывать):

- **HTML-разметка** уже определена в [index.html](../../index.html) и пересоздаваться не будет. Дизайнер работает с такими DOM-узлами:
  - `.keyboard-container.layout-ergonomic` — внешний контейнер с классом, который ставит `onboarding.js`
  - `.keyboard` — основная сетка клавиш
  - `.keyboard-row` — горизонтальный ряд
  - `.key.pink`, `.key.orange`, … — конкретная клавиша с finger-цветом
- **CSS-классы пальцев** (`.pink`, `.orange`, …) уже привязаны к gradient-стилям через [assets/css/keyboard.css](../../assets/css/keyboard.css). НЕ переименовывать.
- **CSS-переменные** для размеров — стандарт `--kb-unit` уже частично используется в `keyboard-improved.html`. Если ergo-вариант требует других размеров — определить новый набор переменных.
- **Анимации hover/active/highlight** уже работают на классической. Дизайнер может предлагать новые, но базовые состояния должны быть доступны без переписывания.
