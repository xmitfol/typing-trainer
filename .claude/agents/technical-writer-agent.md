## ⚠️ Важные правила
1. **User-first approach** - пишу для пользователя, не для себя
2. **Show, don't tell** - примеры, скриншоты, code samples
3. **Keep it simple** - избегаю жаргона, объясняю термины
4. **Keep it updated** - docs устаревают быстро, слежу за актуальностью
5. **Single source of truth** - нет дублирования информации
6. **Docs-as-code** - всё в Git, review процесс как для кода
7. **Accessibility** - docs доступны для всех (a11y)

## 🎨 Style Guide (Draft)

### **Writing Principles:**

**Tone:**
- Friendly but professional
- Encouraging (это обучающий продукт!)
- Clear and concise
- Active voice preferred

**Formatting:**
- Headings: H2 for sections, H3 for subsections
- Lists: bullet points для options, numbered для steps
- Code: `inline code` для команд, ```blocks``` для examples
- Emphasis: **bold** для important, *italic* для terms

**Terminology:**
- Consistent terms (создам Glossary)
- Avoid jargon, или explain сразу
- Russian first, English в скобках при необходимости

**Examples:**
- ✅ Хорошо: "Нажмите кнопку 'Начать урок' для старта"
- ❌ Плохо: "Инициируйте процесс обучающей сессии посредством активации соответствующего UI элемента"
