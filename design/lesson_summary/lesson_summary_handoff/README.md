# Отчёт по уроку (lesson summary) — верстка для программиста

Богатый финальный экран конца УРОКА (экран B из твоего документа). Эталон-визуал — `vanilla/lesson-summary.html` + `assets/js/lesson-summary.js`. Два состояния переключаются тумблером вверху (в проде состояние выбирается по статистике захода).

## Где это в потоке
- **Конец ШАГА** (упражнение) — оставить кратким (`✓ Шаг пройден`), как сейчас.
- **Конец УРОКА** (после финального захода) — этот богатый отчёт.
- Поток: реализуем **Вариант 2** — на последнем шаге кнопка ведёт «Финальный заход →», а не «Назад к уроку». После финального захода → этот отчёт.

## Структура (сверху вниз)
1. **Шапка** — портрет наставника (130px, в пастельном круге) + kicker «УРОК ПРОЙДЕН ✓» (успех) / «УРОК ЗАВЕРШЁН» (с трудом) + «Урок N» + название урока.
2. **«Ты освоил»** — новые клавиши урока как кейкапы **в цвет своего пальца** (фон `${finger}22`, бордер + текст — тёмный оттенок пальца). Подпись «+N букв · всего X из 33».
3. **Метрики** — горизонтальный ряд, разделён линиями свер/снизу:
   - **Точность — главная** (48px, цвет: ≥90 зелёный / ≥75 оранжевый / <75 красный).
   - Оценка (звёзды, поддержка половинок).
   - Ритмичность (26px).
   - **Скорость скрыта на ранних уроках** (`showSpeed:false`). С урока 6 — показывать мелко.
4. **Разбор** — 2 колонки: «💪 Что хорошо» (зелёный заголовок) / «🎯 Над чем поработать» (оранжевый). Если ошибок нет — правая колонка «Ошибок почти нет — отлично!».
5. **Мотивация** — реплика наставника в Serif-курсиве, плашка в цвет состояния (синяя успех / жёлтая поддержка) + «— Имя, наставник».
6. **Прогресс курса** — «Урок N / 20» + полоса.
7. **Действия** — успех: первична «Следующий урок →»; с трудом: первична «↻ Повторить урок» (следующий — вторичен). Всегда «К списку уроков» третичной.

## Два состояния (по статистике захода)
| | success | struggle |
|---|---|---|
| Конфетти-полоса сверху | да | нет |
| Тон kicker | праздничный | нейтральный |
| Цвет плашки мотивации | синяя | жёлтая |
| Первичная кнопка | Следующий урок | Повторить урок |
| Мотивация | «успех» | «поддержка» (без давления) |

Порог: например ошибок > N% или звёзд < 3 → struggle. Точный порог — за продуктом.

## Данные (payload отчёта)
```js
{
  mentor: 'maxim', mentorName: 'Максим',
  lessonNum: 1, lessonTitle: 'Первые буквы: А, О, В, Л',
  keys: [['а','blue'],['о','indigo'],['в','green'],['л','green']],  // [буква, палец]
  learnedTotal: 4, learnedAll: 33,
  stars: 5, acc: 98, rhythm: 84, speed: 142, showSpeed: false,
  good: ['...'], work: ['...'],          // из coach.js (уже есть)
  quote: '...',                          // мотивация наставника (coach.js)
  lesson: 1, lessonsTotal: 20,
  state: 'success' | 'struggle',
}
```

## Что переиспользовать (уже в коде)
- `coach.js` — «что хорошо / над чем поработать» + топ-ошибка по клавише + мотивация по наставникам. Прокинуть в `good`/`work`/`quote`.
- Палитра пальцев — токены `--f-*` (кейкапы).
- Портреты — `portraits.mentor(id, size)`.
- Конфетти/звёзды — стили в `lesson-summary.html`.

## Открытые вопросы (мои ответы дизайнера)
1. **Скорость на ранних уроках** — скрыть на 1-5, с 6 показывать мелко. Главная метрика — точность.
2. **Отдельный экран или расширение `#success`** — расширить `#success` (один компонент, режим «отчёт урока» vs «шаг пройден»).
3. **Поток** — Вариант 2 (последний шаг → «Финальный заход →»).
4. **Отчёт по блоку** (каждые ~5 уроков) — нужен как веха, но отдельной задачей, не блокирует.
