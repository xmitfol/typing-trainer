# Правки экрана тренажёра (task.html) — для программиста

Ниже список правок по текущей реализации `task.html?tier=…&lesson=…`.
Эталон с уже внесёнными правками — в этом же пакете (`task.html` + `assets/`). Можно сверяться/подтягивать.

## Что уже исправлено в эталоне (подтяни к себе)

1. **Окно набора больше не пустует снизу.** Было: фикс. высота 200px → текст в одну строку, снизу дыра. Стало: `min-height:90px; max-height:220px`, контент центрирован по вертикали (`display:flex; align-items:center; flex-wrap:wrap`).

2. **Текущий символ подсвечивается в цвет пальца.** Текущая буква в тексте получает `data-finger="..."` и красится в насыщенный цвет своего пальца (Ф=мизинец розовый, Ы=безымянный оранжевый и т.д.). Взгляд связывает «что печатать» → «каким пальцем» → «какая клавиша подсвечена». Карта `CHAR_FINGER` в `task.js`.

3. **Совет наставника не наезжает на ✕.** Bubble сдвинут левее (`top:-88px; right:60px`), крестик закрытия свободен.

4. **Тулбар: красные бейджи только на выключенном.** По умолчанию: подсветка пальцев ВКЛ (без бейджа), звук ВЫКЛ (бейдж), метроном ВЫКЛ (бейдж). Не вешать бейдж на включённые функции.

## Что доделать на стороне реализации

5. **График скорости.** В твоей версии под окном набора — пустая сетка 100/300/600 без линии (выглядит сломанным блоком). Варианты:
   - рисовать линию по мере набора (`<canvas>` + точка на каждый интервал `updateStats`), или
   - скрывать график до старта печати и показывать по первому нажатию, или
   - убрать совсем, если не критичен.
   В эталоне графика нет — добавь только если нужен продукту.

6. **Иконки тулбара — добавить tooltips/подписи.** Сейчас иконки мелкие, без подписей — непонятно что есть что. На каждую кнопку `title="..."` (в эталоне уже проставлены). Идеально — короткая подпись под иконкой.

7. **Текст урока «дд» / «вода дала овал».** Проверь данные в `lesson-data.js` — «дд» в начале выглядит как опечатка. Контент уроков согласовать с методикой.

## Дефолты (не менять)
- Подсветка пальцев — ВКЛ
- Звук нажатия — ВЫКЛ
- Метроном — ВЫКЛ
- Масштаб — 100% (степпер −/+ по 10%, 70–150%, масштабирует ВСЮ карточку через `.task-card { zoom }` — режим для слабовидящих)

## Маршрут
`task.html` — только из `lesson.html` (теория) по кнопке «Начать упражнение». success → «Продолжить» → следующий урок через `lesson.html`. Не делать task точкой входа из списка.

## Технические замечания
- Клавиатура `<typing-keyboard>` — в Shadow DOM. Проверять живым браузером (скриншот-тулзы её не видят). После правок JS — хард-рефреш (кэш Web Component).
- Палитра пальцев и деление рук ЙЦУКЕН — фиксированы, не менять.
